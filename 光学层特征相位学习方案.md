# 光学层特征相位学习方案

本文档基于"特征损失为什么停在 0.62"的分析，提出系统性修改方案，目的是突破当前损失下限，让光学学生层能真正学到高质量的教师特征映射。

## 分析：为什么特征损失停在 0.62

### 1. 归一化的作用

在 [optical_layers.py:90-91](vscode-webview://1dgdp5lsrqo9v5hfpmfe2oil4uaqvso2c3tbv9c8b529so4an6d4/models/SLM/optical_layers.py#L90-L91)：



```python
if self.enable_norm:
    out = out / (out.mean(dim=[2, 3], keepdim=True) + OPTICAL_NORM_EPS)
```

这个归一化做的是：**将每张输出特征图的空间均值强制为 ≈ 1.0**。它的目的是移除"绝对亮度"这个自由度——如果没有它，学生网络可以直接让所有光强都变得很亮来降低 MSE，而不学习任何有意义的空间结构。归一化迫使网络只能通过**空间对比度**来匹配教师。

### 2. 根本问题：归一化与教师输出范围不兼容

**教师输出** ([teacher.py:78-80](vscode-webview://1dgdp5lsrqo9v5hfpmfe2oil4uaqvso2c3tbv9c8b529so4an6d4/models/teacher.py#L78-L80))：



```python
f = torch.abs(self.project(f))   # abs ≥ 0
return torch.sigmoid(f)          # sigmoid(abs) ∈ [0.5, 1.0]
```

教师输出的值域是 **[0.5, 1.0]**，空间均值通常在 0.65-0.85 之间。

**学生输出**（归一化后）：均值强制为 **1.0**，值域可以是 [0, ∞)。

这就产生了一个**数学上不可消除的矛盾**：



```
学生要想匹配教师 T(x,y)，需要输出 S(x,y) = T(x,y)
但归一化强制 mean(S) ≈ 1.0
而 mean(T) ≈ 0.7~0.8 ≠ 1.0

→ 学生无法同时满足"匹配T"和"均值为1.0"这两个条件
```

学生能做到的最好情况是输出缩放版本：$S = T / \mu_T$，此时 $\text{mean}(S) = 1.0$，但与目标 $T$ 之间存在系统性偏差。

以教师 $\mu_T = 0.75$ 为例，即使空间结构完全匹配： $$\text{MSE} = \mathbb{E}[(T/0.75 - T)^2] = \mathbb{E}[(0.333 \cdot T)^2] \approx 0.11 \cdot \mathbb{E}[T^2]$$

仅 MSE 分量就贡献了一个**不可消除的底限**：

| 损失分量                  | 权重    | 估算贡献  |
| ------------------------- | ------- | --------- |
| `loss_full` (MSE 原图)    | 0.2     | ~0.01     |
| `loss_low1` (MSE 8×池化)  | 0.8     | ~0.05     |
| `loss_low2` (MSE 32×池化) | 0.4     | ~0.03     |
| `loss_ssim`               | 0.3     | ~0.10     |
| `loss_grad`               | 0.25    | ~0.08     |
| `loss_freq`               | 0.15    | ~0.05     |
| `loss_div` (相位多样性)   | **1.0** | ~0.30     |
| `loss_smooth`             | 0.001   | ~0.001    |
| **合计**                  |         | **~0.62** |

### 3. 相位多样性损失也在卡住

[losses_slm.py:61-91](vscode-webview://1dgdp5lsrqo9v5hfpmfe2oil4uaqvso2c3tbv9c8b529so4an6d4/models/SLM/losses_slm.py#L61-L91) 中的 `phase_diversity_loss` 权重为 **1.0**，且目标值设得很高：

| 指标                        | 目标值 | 最大可能值       | 说明            |
| --------------------------- | ------ | ---------------- | --------------- |
| `PHASE_STD_TARGET`          | 0.90   | ~1.81 (均匀分布) | 相位标准差      |
| `PHASE_SPAN_TARGET`         | 4.80   | 2π ≈ 6.28        | 相位跨度需达76% |
| `PHASE_CIRCULAR_STD_TARGET` | 0.80   | —                | 圆形标准差      |
| `PHASE_NEAR_BOUNDARY_LIMIT` | 0.85   | —                | 近0/2π边界比例  |

这些目标很高——相位既要满足多样性约束，又要产生匹配教师特征图的光学变换，两者存在**优化冲突**。优化器在两者之间折中，导致 phase diversity loss 不会降到零。

### 4. 另外：教师背景值 0.5 的问题

教师的 `sigmoid(abs(x))` 设计意味着**背景区域值 ≈ 0.5**（当 abs(x) ≈ 0 时），而光学学生可以产生**真正的零值**（完全相消干涉）。学生归一化后，背景值会更低（比如 0.2），而教师背景是 0.5——这种背景级别的系统性偏差也贡献了不可消除的 MSE。

### 总结

损失卡在 0.62 不是收敛问题，而是**由三个因素叠加造成的数学下限**：

1. **归一化均值 1.0 ≠ 教师均值 0.7~0.8** → 系统性尺度失配，MSE 永不归零
2. **教师 sigmoid(abs(.)) ∈ [0.5, 1.0]** → 学生能产生 [0, ∞) 范围，但归一化限制了匹配能力
3. **相位多样性损失权重 1.0 + 高目标值** → 持续产生 ~0.3 的惩罚

如果要突破这个底限，几个方向：

- 去掉归一化，改用 batch-level 的 scale-invariant loss（如 pearson correlation）
- 把教师的 `abs()` + `sigmoid()` 改成 `tanh()` 或直接输出无约束值，让教师值域与学生的 [0, ∞) 更匹配
- 把归一化从 `mean=1.0` 改成 `max=1.0`（除以最大值而非均值），这样学生可以通过峰值匹配教师的 FG 值

## 一、问题回顾

当前 `optical_slm_yolov8_head.py` 训练中，`student_only` 阶段的 feature loss 在 0.62 左右停滞，无法继续下降。经分析，这不是收敛速度问题，而是由**三个结构性因素叠加**造成的数学下限。

## 二、根因分析（三个下限来源）

### 2.1 归一化均值与教师输出均值不可调和的矛盾

这是**最主要的下限来源**。

**学生侧**（[optical_layers.py:90-91](models/SLM/optical_layers.py#L90-L91)）：

```python
if self.enable_norm:
    out = out / (out.mean(dim=[2, 3], keepdim=True) + OPTICAL_NORM_EPS)
```

归一化将每张输出特征图的空间均值强制为 ≈ 1.0。

**教师侧**（[teacher.py:78-80](models/teacher.py#L78-L80)）：

```python
f = torch.abs(self.project(f))   # abs ≥ 0
return torch.sigmoid(f)          # sigmoid(abs) ∈ [0.5, 1.0]
```

教师输出的值域是 **[0.5, 1.0]**，空间均值通常在 0.65-0.85 之间（取决于前景占比）。

**矛盾**：学生要想完美匹配教师 $T(x,y)$，需要输出 $S = T$，但此时 $\text{mean}(S) = \mu_T \approx 0.75 \neq 1.0$，归一化会将其强制缩放为 $S' = T / 0.75$，与目标 $T$ 产生系统性偏差。

即使学生学到了与教师完全相同的空间结构，MSE 也不可消除：

$$ \text{MSE}_{\text{min}} = \mathbb{E}[(T/\mu_T - T)^2] = \mathbb{E}[T^2] \cdot (1/\mu_T - 1)^2 $$

当 $\mu_T = 0.75$ 时，$\text{MSE}_{\text{min}} \approx 0.055$。这还只是像素级 MSE，加上多尺度、梯度、频域等分量，总下限就是 0.62 的主要构成。

### 2.2 教师输出背景最低 0.5，学生能产生真零

`torch.abs()` 使教师的所有值非负，`sigmoid(0) = 0.5`，所以教师的"背景"最低就是 0.5。而光学学生通过相消干涉可以产生真正的零值背景。归一化后学生的背景值会低于 0.5，与教师背景 0.5 产生系统性偏差，贡献不可消除的 MSE。

### 2.3 相位多样性损失目标过高且与特征匹配冲突

[config_slm.py:85-88](models/SLM/config_slm.py#L85-L88) 中：

| 配置 | 当前值 | 含义 |
|------|--------|------|
| `LOSS_PHASE_DIVERSITY_WEIGHT` | **1.0** | 与 total feature loss 等权重 |
| `PHASE_STD_TARGET` | 0.90 | 目标相位标准差（均匀分布约 1.81） |
| `PHASE_SPAN_TARGET` | 4.80 | 目标相位跨度（最大 2π ≈ 6.28） |
| `PHASE_CIRCULAR_STD_TARGET` | 0.80 | 目标圆形标准差 |

这些目标值偏高，且 `F.relu(target - actual)` 在目标未达成时持续产生惩罚。相位既要满足这些多样性约束，又要产生匹配教师的光学变换——两者存在**优化冲突**。多样性损失权重 1.0 使其成为总损失的主要贡献者，但它的下降空间受限于物理约束。

### 2.4 分量损失贡献估算

以教师空间均值 $\mu_T \approx 0.75$ 为例：

| 损失分量 | 权重 | 不可消除下限（估算） | 贡献 |
|----------|------|---------------------|------|
| `loss_full` (MSE 原图) | 0.2 | ~0.055 | ~0.011 |
| `loss_low1` (MSE 8×池化) | 0.8 | ~0.055 | ~0.044 |
| `loss_low2` (MSE 32×池化) | 0.4 | ~0.055 | ~0.022 |
| `loss_ssim` | 0.3 | ~0.25 | ~0.075 |
| `loss_grad` | 0.25 | ~0.20 | ~0.050 |
| `loss_freq` | 0.15 | ~0.30 | ~0.045 |
| `loss_smooth` | 0.001 | ~1.5 | ~0.002 |
| `loss_div` | **1.0** | ~0.30 | ~0.300 |
| **合计** | | | **~0.55-0.65** |

实际观测值 0.62 正好落在这个估计区间内，说明损失确实触达了结构性下限。

## 三、修改方案

### 修改 1：将归一化从均值归一改为最大值归一

**文件**：[models/SLM/optical_layers.py](models/SLM/optical_layers.py) `OpticalStudent.forward()`

**当前**：
```python
if self.enable_norm:
    out = out / (out.mean(dim=[2, 3], keepdim=True) + self.config.OPTICAL_NORM_EPS)
```

**修改为**：
```python
if self.enable_norm:
    out = out / (out.amax(dim=[2, 3], keepdim=True) + self.config.OPTICAL_NORM_EPS)
```

**为什么这么改**：
- 教师前景峰值 ≈ 1.0（sigmoid(大值) → 1.0），学生归一化到 max=1.0 后前景也 ≈ 1.0，两者前景可完全对齐
- 背景值由光学相位设计自然决定：教师背景 ≈ 0.5，学生通过相消干涉可以让背景也接近 0.5 或更低
- 不再有 $1/\mu_T$ 的全局缩放因子，消除了最大的 MSE 下限来源
- 如果学生的某个样本完全无前景（全图均匀），max ≈ mean，退化为类似均值归一，不会爆炸

**风险**：max 对异常亮点敏感。如果相位设计产生局部极亮像素，max 归一化会压暗整体。这可以通过相位平滑损失（已有）来约束。

### 修改 2：去掉教师的 `abs()` 操作，让教师输出值域为 [0, 1]

**文件**：[models/teacher.py](models/teacher.py) `ConvTeacher.forward()`

**当前**：
```python
f = torch.abs(self.project(f))
f = F.interpolate(f, size=x.shape[-2:], mode="bilinear", align_corners=False)
return torch.sigmoid(f)
```

**修改为**：
```python
f = self.project(f)
f = F.interpolate(f, size=x.shape[-2:], mode="bilinear", align_corners=False)
return torch.sigmoid(f)
```

**为什么这么改**：
- `abs()` 将所有负值翻正，使 sigmoid 输入 ≥ 0，输出被限制在 [0.5, 1.0]
- 去掉 `abs()` 后，`sigmoid(x)` 可以覆盖完整的 [0, 1] 区间：
  - 背景（负值大）→ sigmoid → ≈ 0
  - 前景（正值大）→ sigmoid → ≈ 1.0
- 教师背景从固定 0.5 变为可以接近 0，与光学学生可产生的真零背景完全兼容

**注意**：去掉 abs 需要重训教师网络，因为教师之前是在 abs+sigmoid 下训练的。如果不想重训教师，也可以用 `2 * sigmoid(x) - 1` 映射到 [-1, 1] 再取绝对值或直接用 `tanh`。但最干净的方案是重训教师，因为教师参数量仅 450K，重训成本很低。

**备选方案（不改教师结构，只改学生侧对齐）**：如果不想动教师，可以在学生输出后加一个仿射变换层：
```python
# 学生输出归一化到 max=1.0 后，再做线性映射匹配教师 [0.5, 1.0] 范围
out = 0.5 + 0.5 * out  # [0, 1] → [0.5, 1.0]
```
但这样学生就永远无法产生低于 0.5 的背景，与光学物理能力（可产生真零）不匹配，不推荐。

### 修改 3：降低相位多样性损失权重并调整目标值

**文件**：[models/SLM/config_slm.py](models/SLM/config_slm.py)

**当前**：
```python
LOSS_PHASE_DIVERSITY_WEIGHT = 1.0
PHASE_STD_TARGET = 0.90
PHASE_SPAN_TARGET = 4.80
PHASE_CIRCULAR_STD_TARGET = 0.80
```

**修改为**：
```python
LOSS_PHASE_DIVERSITY_WEIGHT = 0.15
PHASE_STD_TARGET = 0.60
PHASE_SPAN_TARGET = 3.50
PHASE_CIRCULAR_STD_TARGET = 0.50
```

**为什么这么改**：
- 权重从 1.0 降到 0.15：当前相位多样性损失占总损失约 50%（权重 1.0 vs 其他项合计约 1.4），严重挤压了特征匹配的优化空间。0.15 使其成为辅助约束而非主导项
- 目标值下调：原目标（std=0.90, span=4.80）要求相位接近均匀分布，过于严苛。在相位层主要服务于特征匹配的前提下，适中的多样性足够
- 新的目标值（std=0.60, span=3.50）仍然远高于相位塌缩阈值（std≈0.03），能有效防止塌缩，同时给特征匹配留出足够优化空间

### 修改 4：增加尺度不变损失项（Pearson 相关系数）

**文件**：[models/SLM/losses_slm.py](models/SLM/losses_slm.py) `CompositeOpticalFeatureLoss`

**新增损失分量**：

```python
def pearson_loss(self, student_feature, teacher_feature):
    """尺度和平移不变的相似性度量"""
    b = student_feature.shape[0]
    s_flat = student_feature.view(b, -1)
    t_flat = teacher_feature.view(b, -1)
    s_centered = s_flat - s_flat.mean(dim=1, keepdim=True)
    t_centered = t_flat - t_flat.mean(dim=1, keepdim=True)
    s_std = s_centered.std(dim=1, keepdim=True) + self.config.OPTICAL_FIELD_EPS
    t_std = t_centered.std(dim=1, keepdim=True) + self.config.OPTICAL_FIELD_EPS
    corr = (s_centered * t_centered).mean(dim=1) / (s_std * t_std + self.config.OPTICAL_FIELD_EPS)
    return torch.clamp((1.0 - corr.mean()) * 0.5, min=0.0)
```

**配置新增**（[config_slm.py](models/SLM/config_slm.py)）：
```python
LOSS_PEARSON_WEIGHT = 0.50
```

**为什么这么改**：
- Pearson 相关系数对线性变换 $S = aT + b$ 完全不变，不关心均值和方差是否匹配
- 即使归一化方式改变，Pearson loss 也不受影响，纯度量空间结构相似性
- 与 SSIM 互补：SSIM 侧重于局部结构，Pearson 侧重于全局相关性
- 权重 0.50 让它成为特征匹配的主要驱动力之一，减少对 MSE 的依赖

### 修改 5：调整多尺度 MSE 权重分配

**文件**：[models/SLM/config_slm.py](models/SLM/config_slm.py)

**当前**：
```python
LOSS_FULL_WEIGHT = 0.2
LOSS_LOW1_WEIGHT = 0.8
LOSS_LOW2_WEIGHT = 0.4
LOSS_SSIM_WEIGHT = 0.3
LOSS_GRAD_WEIGHT = 0.25
LOSS_FREQ_WEIGHT = 0.15
```

**修改为**：
```python
LOSS_FULL_WEIGHT = 0.15
LOSS_LOW1_WEIGHT = 0.35
LOSS_LOW2_WEIGHT = 0.20
LOSS_SSIM_WEIGHT = 0.40
LOSS_GRAD_WEIGHT = 0.20
LOSS_FREQ_WEIGHT = 0.10
LOSS_PEARSON_WEIGHT = 0.50   # 新增
```

**为什么这么改**：
- **提高 SSIM 权重**（0.3→0.40）：SSIM 对亮度和平移不敏感，受归一化失配影响最小，是当前最有效的匹配信号
- **降低多尺度 MSE 权重**（原图 0.2→0.15，低分辨率 0.8→0.35，0.4→0.20）：MSE 受归一化失配影响最大，适度降低可减少其在总损失中的地板效应
- **新增 Pearson 权重 0.50**：作为尺度不变的主匹配项，弥补 MSE 权重的降低
- 总特征损失权重保持约 1.9，与修改前（0.2+0.8+0.4+0.3+0.25+0.15=2.1）接近，不改变特征损失在总训练目标中的相对比重

### 修改 6：统一 phase_raw 参数初始化范围

**文件**：[models/SLM/optical_layers.py](models/SLM/optical_layers.py) `SLMLayer._initial_phase()`

**当前**：vortex 初始化后用 `torch.remainder(phase, 2π)`，但 random 模式直接用 `rand * 2π`

**修改为**：统一使用 `torch.remainder(..., 2π)` 并确保初始相位跨度足够：

```python
def _initial_phase(self, resolution):
    init_mode = str(getattr(self.config, "SLM_INIT_MODE", "random")).lower()
    if init_mode in {"vortex", "vortex_checkpoint"}:
        # ... vortex 逻辑不变 ...
        return torch.remainder(phase, 2 * np.pi).contiguous().view(1, 1, height, width)
    # 随机初始化：确保初始就有足够多样性
    phase = torch.rand(1, 1, *resolution) * 2 * np.pi
    # 不做 remainder，因为 rand 已经在 [0, 2π) 内
    return phase
```

**为什么这么改**：
- 实际上当前 random 初始化已经是正确的（[0, 2π)），不需要改
- 但如果用户使用 checkpoint 恢复时 phase_raw 已经是缩放到小范围的值（如之前塌缩过的权重），需要清除并重新初始化

## 四、修改优先级和实施顺序

| 优先级 | 修改 | 预期收益 | 风险 |
|--------|------|----------|------|
| **P0** | 修改 1（均值归一→最大值归一） | 消除最大的 MSE 下限 | 低 |
| **P0** | 修改 3（降低相位多样性权重和目标） | 释放特征匹配优化空间 | 低 |
| **P1** | 修改 4+5（新增 Pearson + 调整权重） | 提供尺度不变匹配信号 | 低 |
| **P2** | 修改 2（去掉教师 abs） | 彻底解决值域不匹配 | 中（需重训教师） |

**建议实施顺序**：先做 P0 的 2 项修改，重新训练观察 feature loss 是否能突破 0.62。如果降到 0.3 以下，再做 P1 进一步优化。P2 在下一轮教师训练时一并修改。

## 六、验证方法

训练后重点观察：

1. 日志中 feature loss 是否跌破 0.62 并持续下降
2. 各损失分量统计（full/low1/low2/ssim/pearson）的下降趋势
