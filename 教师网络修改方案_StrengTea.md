# 教师网络修改方案

## 1. 目标

当前 `optical_teacher_yolo.py` 的训练方式调整为真正的三阶段联合训练，但三个阶段都保持：

- 教师网络参与训练
- 检测头参与训练
- 不再采用“后期冻结检测头、只继续塑形教师特征”的旧路线

这样做的目标是把训练过程拆成更清晰的三个任务：

1. 第一阶段先把目标位置找稳，尽量对准 GT，减少一个目标很多框。
2. 第二阶段继续训练教师网络和检测头，但明显加强教师特征学习，让教师特征不只会“亮在位置上”，还要学到目标内部纹理、边缘和类别区分信息。
3. 第三阶段用更低学习率做联合平衡，提升 `mAP`、预测稳定性和最终可视化质量。

## 2. 可行性判断

这条路线是可行的，而且比“后期只改教师、不让检测头跟着适配”更稳。

原因很直接：

- 你的最终使用方式本来就是教师网络输出单通道特征，再接检测头。
- 如果后期只改教师，不改检测头，就会出现输入分布漂移，检测头接不住的问题。
- 现在三个阶段都让教师和检测头一起训练，就能避免这个割裂。

其中你提到的“第二阶段扩大 10 倍教师学习”，直接把整体学习率乘 10 风险太大，容易把训练拉崩。

因此本次实现采用的是更稳的落地方式：

- 第二阶段把教师塑形监督强度放大到第一阶段的 `10x`
- 同时适度提高教师分支学习率
- 检测头保持继续学习，但学习率比教师更保守

这样更符合“加强教师学习”的真实目的，也更适合当前结构。

## 3. 三阶段训练设计

## 3.1 第一阶段：`locate_gt`

目标：

- 优先让框尽快对准 GT
- 减少一个目标上出现很多碎框
- 让模型先学会“目标在哪”

策略：

- 提高 `box/objectness` 权重
- 压低分类权重
- 教师塑形只给基础强度，避免一开始就让 teacher heatmap 主导训练
- 教师和检测头使用相同学习率联合训练

实现要点：

- `STAGE1_LOCATE_EPOCHS = 40`
- `PHASE1_TEACHER_LR = 4e-4`
- `PHASE1_DETECTOR_LR = 4e-4`
- `PHASE1_TEACHER_GUIDANCE_SCALE = 1.0`

## 3.2 第二阶段：`texture_detail`

目标：

- 在第一阶段已经能大致找准目标位置的基础上
- 继续训练教师网络和检测头
- 明显加强教师特征对纹理、边缘、主体结构和类别区分的学习

策略：

- 分类权重提高
- 教师塑形监督强度扩大到第一阶段的 `10x`
- 教师学习率高于检测头
- 检测头不冻结，继续跟随 teacher 特征共同适配

实现要点：

- `STAGE2_TEXTURE_EPOCHS = 50`
- `PHASE2_TEACHER_LR = 6e-4`
- `PHASE2_DETECTOR_LR = 3e-4`
- `PHASE2_TEACHER_GUIDANCE_SCALE = 10.0`

说明：

- 这里的“10 倍”落在教师塑形监督强度上
- 不是把整个优化器粗暴放大 10 倍

## 3.3 第三阶段：`balance_refine`

目标：

- 平衡教师网络和检测头
- 收敛到更稳定的检测表现
- 提升 `mAP`、预测框质量和最终可视化效果

策略：

- 教师和检测头都继续训练
- 双方学习率都降低
- 分类权重保持相对较高
- 背景抑制略加强，帮助压制重复框和虚警

实现要点：

- `STAGE3_BALANCE_EPOCHS = 30`
- `PHASE3_TEACHER_LR = 1.5e-4`
- `PHASE3_DETECTOR_LR = 1.2e-4`
- `PHASE3_TEACHER_GUIDANCE_SCALE = 2.5`

## 4. 当前代码层面的关键变化

本次 `optical_teacher_yolo.py` 已修改为：

1. 三个阶段都训练教师网络 + 检测头
2. 去掉“后期冻结检测头”的主训练路线
3. 阶段切换时按阶段重建优化器
4. 教师和检测头支持不同学习率
5. 教师塑形损失按阶段切换强度
6. 检测损失权重也按阶段自动切换

## 5. 现在的阶段调度逻辑

当前 `Config.get_stage_settings(epoch)` 会返回每个 epoch 的阶段配置，包含：

- `phase`
- `box_weight`
- `obj_weight`
- `noobj_weight`
- `cls_weight`
- `size_weights`
- `teacher_lr`
- `detector_lr`
- `teacher_guidance_scale`

训练主循环会在阶段变化时：

1. 切换阶段名
2. 重建优化器
3. 应用新的 teacher/detector 学习率
4. 用新的阶段系数计算检测损失和教师塑形损失

## 6. 为什么这版比旧方案更适合现在的任务

旧方案的问题是：

- 前期 teacher + detector 联合训练
- 后期冻结 detector
- 只继续改 teacher

这样很容易出现：

- teacher 特征继续变化
- detector 已经不再适配新特征
- 结果是后期看起来 teacher 更“有图案”，但检测表现反而不稳定

新方案改成三阶段全联合后：

- 第一阶段先稳住定位
- 第二阶段再放大 teacher 学习去补纹理和类别信息
- 第三阶段让 teacher 和 detector 一起收敛

更符合你现在这条链路的真实需要。

## 7. 运行后重点看什么

重新训练后，建议重点观察这几项：

1. 第一阶段结束时：
   - 重复框是否减少
   - 大目标是否更容易整体框住
   - GT 附近是否仍然有大量偏框

2. 第二阶段中后段：
   - 教师特征图是否不再只是位置亮斑
   - 目标内部纹理和边缘是否更明显
   - 分类是否开始比第一阶段更稳定

3. 第三阶段：
   - `mAP50` 是否比第二阶段更稳
   - 背景误检是否进一步下降
   - 一个目标多框的问题是否继续缓解

## 8. 当前建议

这版方案已经适合作为当前主线继续训练。

如果下一轮结果仍然主要表现为：

- 一个大目标很多小框
- 背景误检很多
- 大目标只能框住一部分

那么下一步优先继续看的不是网络接口，而是：

1. 第二阶段的 teacher guidance 是否仍然偏强或偏弱
2. 第二阶段分类权重是否还需要再提高
3. 第三阶段的背景抑制是否还要再加强
4. 可视化阈值和评估阈值是否需要进一步分离

## 9. 多 GPU 与单 GPU 运行

当前 `optical_teacher_yolo.py` 已支持自动按可见 GPU 数运行：

- 2 张或更多 GPU 时：
  - 默认自动使用全部可见 GPU
  - 教师网络 + 检测头联合模型会自动用 `DataParallel` 包装
- 只有 1 张 GPU 时：
  - 自动退回单卡
- 没有 GPU 时：
  - 自动退回 CPU

因此你在当前有 2 张 `RTX A6000` 的服务器上运行时，不需要再手动改成双卡版本；换到只有 1 张卡的服务器，也不需要额外删改代码。
# 教师网络修改方案 StrengTea

## 2026-04-30 增补

这次在当前 `StrengTea` 主线代码上，补入了来自 `enhance` 分支里“更偏工程稳定性和后处理约束”的部分思路，但没有照搬 `enhance` 里那套旧权重参数。

本轮新增的核心点如下：

1. 训练轮数扩展到 `800 epoch`
2. 正样本进一步收紧：
   - `POSITIVE_ANCHOR_IOU = 0.35`
   - `MAX_POSITIVE_ANCHORS = 1`
3. 背景抑制进一步增强：
   - `NOOBJ_IGNORE_IOU = 0.68`
   - `CONF_THRESH = 0.66`
   - `NMS_THRESH = 0.16`
4. 保留并加强碎框抑制：
   - `ENABLE_CONTAINMENT_SUPPRESSION = True`
   - `CONTAINMENT_SUPPRESS_RATIO = 0.90`
   - `CONTAINMENT_AREA_RATIO_LIMIT = 0.50`
5. 新增同类框保守融合：
   - 开启 `ENABLE_WBF = True`
   - 只对同类别、IoU 足够高、中心点足够近、尺寸差不大的框做融合
   - 如果偏差明显，则不融合，避免把两个独立目标错误并成一个框
6. 可视化默认改为 `matched-only`
   - 预测图只显示和 GT 达到最低 IoU 匹配条件的框
   - 后台真实评估和后处理仍然照常进行，不是简单“只画不算”
   - 这样做的目的是让训练过程中的可视化更干净，避免满屏碎框干扰判断
7. 加速与加载器参数增强：
   - `NUM_WORKERS = min(12, os.cpu_count() or 0)`
   - `PERSISTENT_WORKERS = True`
   - `PREFETCH_FACTOR = 4`
   - `PIN_MEMORY = True`
   - `ENABLE_CUDNN_BENCHMARK = True`
   - `ENABLE_TF32 = True`
   - `ENABLE_CHANNELS_LAST = True`

## 本轮修改意图

这轮不是继续从损失权重本身大幅摆动，而是优先从以下三层入手提升 `mAP` 和观感一致性：

1. 训练分配层：
   - 让正样本更收敛到“最像 GT 的那个位置”
   - 少给大目标分出多个相邻正样本
2. 推理后处理层：
   - 先压掉明显被包裹的小碎框
   - 再对同类高重叠且几何一致的框做保守融合
3. 可视化层：
   - 只展示 matched-only 结果，减少“看起来很乱但其实只是低质候选框”的干扰

## 关于同类框融合

这里加的不是无条件 WBF，而是“保守同类融合”：

1. 只在同类别内部处理
2. 只有 IoU 达到阈值、中心点距离足够近、宽高比例差不大时才融合
3. 融合后的框坐标按置信度加权平均
4. 如果候选框偏差太大，则保持原始高分框，不强行融合

这样做是为了缓解：

1. 一个大目标上出现多个相近小框
2. 同一主体被多个轻微偏移框重复覆盖

同时尽量避免：

1. 相邻两个真实目标被误合并
2. 大框和明显错位框融合后反而更偏离 GT

## 关于 matched-only 可视化

`matched-only` 只影响训练过程中保存出来的预测图显示，不改变训练损失本身，也不替代正常验证指标。

具体行为：

1. 先跑正常解码、NMS、containment suppression、同类框融合
2. 再在可视化阶段，把预测框和当前图像的 GT 做一次低阈值匹配
3. 只把达到最小 IoU 要求的预测框画出来

这样可以更容易看出：

1. 模型有没有真的贴近 GT
2. 同一目标的重复框有没有下降
3. 背景误检是不是还在“混进去”

而不是被一堆明显不匹配的候选框把图挤满。
