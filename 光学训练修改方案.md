# 光学训练修改方案

本文档记录 `optical_teacher.py` 这一轮为解决“教师检测正常，但学生光学层接检测头后几乎预测为 0”而做的训练路线修改，方便在其他分支同步。

## 一、问题现象

- 教师特征 + 检测头可以正常预测，并有一定 `mAP`。
- 光学学生层训练完成后，使用 `Optical_SLM_yolo_model.py` 加载 `optical_student_best.pth + detector_best.pth` 预测时，结果几乎全 0。
- 可视化里常见现象是：
  - 几乎没有有效框；
  - 偶尔出现细长线状框；
  - 分类和定位都明显失效。

## 二、根因分析

根因不是“检测头坏了”，而是训练路径不闭环：

- 原来的 `optical_teacher.py` 主要在做：
  - 灰度输入 -> 光学学生层 -> 拟合教师特征；
  - 再用一个冻结检测头做 `response map` 约束。
- 但这个过程中，真正用于最终推理的检测头并没有被充分训练到“适应学生特征域”。
- `Response loss` 很低，只能说明响应图整体分布有些接近，不代表：
  - 框回归已经可用；
  - 分类头已经适配；
  - 背景抑制已经学好。

所以最终表现会变成：

- 光学层输出“看起来像有点特征”；
- 但检测头面对 student 域特征时分布不匹配；
- 推理时整体接近失效。

## 三、训练路线改成三阶段

当前修改后，`optical_teacher.py` 改成三阶段训练：

### 1. Phase1: student_only

目标：

- 先让光学学生层稳定拟合教师特征；
- 同时用冻结的教师域检测头做轻量 `response map` 约束。

做法：

- 冻结检测头；
- 只训练 `OpticalStudent`；
- 损失主要由：
  - `feature loss`
  - `response loss`
 组成。

对应配置：

- `PHASE1_STUDENT_EPOCHS`
- `STUDENT_ONLY_LR`
- `FEATURE_LOSS_WEIGHT_PHASE1`
- `RESPONSE_LOSS_WEIGHT_PHASE1`

### 2. Phase2: detector_only

目标：

- 冻结光学层；
- 只训练检测头，让检测头适应 student 输出域。

做法：

- 数据集不再只返回图像和教师特征，也返回 YOLO 标签；
- 在 student 输出特征上，直接计算检测损失；
- 只优化检测头参数。

对应配置：

- `PHASE2_DETECTOR_EPOCHS`
- `DETECTOR_ONLY_LR`
- `DETECTION_LOSS_WEIGHT_PHASE2`

### 3. Phase3: joint_balance

目标：

- 最后低学习率联合微调；
- 让光学层和检测头一起收敛，但重点仍是“稳定适配”，不是大幅改变光学层。

做法：

- 同时解冻学生层和检测头；
- 给学生层更小学习率；
- 给检测头略高学习率；
- 联合优化：
  - `feature loss`
  - `response loss`
  - `detection loss`

对应配置：

- `PHASE3_JOINT_EPOCHS`
- `JOINT_STUDENT_LR`
- `JOINT_DETECTOR_LR`
- `FEATURE_LOSS_WEIGHT_PHASE3`
- `RESPONSE_LOSS_WEIGHT_PHASE3`
- `DETECTION_LOSS_WEIGHT_PHASE3`

## 四、阶段切换时的稳定策略

这是这次修改里很关键的一点。

### 1. Phase1 -> Phase2

不要直接拿 Phase1 最后一轮继续训。

原因：

- Phase1 后期可能已经开始漂移；
- 如果直接用最后一轮进入 Phase2，会把检测头训练建立在一个偏掉的 student 特征上。

当前做法：

- 保存 `optical_student_phase1_best.pth`
- 进入 Phase2 时，默认恢复到 Phase1 的最佳 student 状态

配置：

- `RESTORE_PHASE1_BEST_FOR_PHASE2 = True`

### 2. Phase2 -> Phase3

同样不要直接拿 Phase2 最后一轮继续联合训练。

原因：

- Phase2 后半段检测头可能过拟合 student 当前特征；
- 如果直接拿最后一轮进入联合阶段，容易把 joint 阶段也带偏。

当前做法：

- 保存 phase2 最佳 student/detector 配对
- 进入 Phase3 时，默认恢复到这组最佳配对

配置：

- `RESTORE_PHASE2_BEST_FOR_PHASE3 = True`

## 五、数据集侧修改

`MilitaryFeatureDataset` 不再只返回：

- `gray_image`
- `teacher_feature`

而是返回：

- `gray_image`
- `teacher_feature`
- `targets`

其中 `targets` 为当前图像对应的 YOLO 标签：

- `class_id`
- `x_center`
- `y_center`
- `width`
- `height`

同时新增了 `feature_dataset_collate_fn`，用于 `DataLoader` 拼接：

- 图像 tensor
- 教师特征 tensor
- 目标框列表

这样 Phase2 和 Phase3 才能真正计算检测损失。

## 六、检测适配损失新增内容

新增了 `DetectionAdaptationLoss`，核心作用是：

- 在 student 输出特征上训练检测头；
- 不依赖 `optical_teacher_yolo.py` 导入；
- 在 `optical_teacher.py` 内部独立完成检测损失计算。

包含的核心部分：

- `SigmoidFocalLoss`
- `decode_boxes_to_absolute`
- `bbox_iou_matrix_xywh`
- `bbox_iou_xywh`
- `DetectionAdaptationLoss`

损失项包括：

- `box loss`
- `obj loss`
- `noobj loss`
- `cls loss`

并支持：

- 正样本锚框匹配
- `ignore` 负样本屏蔽
- CIoU 框回归

## 七、本次修复的代码报错

本轮还修复了一个明确的代码缺口：

### 报错位置

`optical_teacher.py` 中：

```python
max_iou = bbox_iou_matrix_xywh(flat_pred_boxes, gt_boxes_abs).max(dim=1).values
```

### 原因

- `bbox_iou_matrix_xywh(...)` 被调用了；
- 但文件里原本没有这个函数定义；
- 因此在运行到这一步时会触发：
  - `NameError`
  - 或者在 IDE 里显示未定义引用报错。

### 已修复

已补充 `bbox_iou_matrix_xywh(boxes1, boxes2)`：

- 输入为两组 `xywh` 格式框；
- 输出为两组框之间的两两 IoU 矩阵；
- 供 `ignore_mask` 逻辑使用。

用途是：

- 对每个预测框，找它和所有 GT 的最大 IoU；
- 若 IoU 超过 `NOOBJ_IGNORE_IOU`，则该位置不再计入 `noobj loss`。

## 八、权重保存策略

除了保存学生权重，现在还会同步保存检测头权重。

### 学生权重

- `optical_student_phase1_best.pth`
- `optical_student_phase2_best.pth`
- `optical_student_best.pth`
- `optical_student_last.pth`
- `optical_student_final.pth`

### 检测头权重

- `detector_phase2_best.pth`
- `detector_best.pth`
- `detector_last.pth`
- `detector_final.pth`

说明：

- `best` 表示联合监控下保存的最佳 student/detector 配对；
- `phase2_best` 表示检测头适配阶段的最佳配对；
- `final` 由 `FINAL_MODEL_SOURCE` 决定是取最佳还是最后一轮。

## 九、日志增强

当前 epoch 日志除了总损失，还会记录检测损失拆分项：

- `Detection`
- `DetBox`
- `DetObj`
- `DetNoObj`
- `DetCls`

这样后续分析时，可以直接判断是：

- 框回归没学到；
- 背景抑制太差；
- 还是分类头塌缩。

## 十、当前建议

如果在另一个分支同步这套改动，建议优先保持以下原则：

- 不改变教师输入输出维度、通道数；
- 不改检测头结构；
- 重点改训练路径，而不是硬改模型形状；
- 一定保留“检测头适配 student 域”这个阶段；
- 一定保留阶段最佳权重恢复逻辑。

## 十一、建议关注的训练结果

重新训练后，重点看以下信号：

- `Phase2` 里 `Detection loss` 是否持续下降；
- `DetNoObj` 是否异常高；
- `Phase3` 后 student + detector 的可视化是否开始出现稳定框；
- `Optical_SLM_yolo_model.py` 的 `mAP50` 是否从接近 0 回升；
- 预测结果是否仍然大量出现线状框、空预测或严重偏框。

如果后续还要继续调参，优先顺序建议是：

1. 先看 `DetNoObj` 是否过高；
2. 再看 `POSITIVE_ANCHOR_IOU` 是否过严；
3. 再看 `Phase2/Phase3` 的轮数比例是否合适；
4. 最后再调 `JOINT_STUDENT_LR`，避免光学层在联合阶段被改得太猛。

## 十二、多 GPU 与单 GPU 运行说明

当前 `optical_teacher.py` 已补充自动设备适配逻辑：

- 如果服务器有 2 张或更多 GPU：
  - 默认自动使用全部可见 GPU
  - 通过 `nn.DataParallel` 包装可训练模块
- 如果服务器只有 1 张 GPU：
  - 自动退回单卡运行
- 如果没有 GPU：
  - 自动退回 CPU

本次实现里，真正影响双卡利用率的关键改动不是只加 `DataParallel`，还包括：

1. 教师网络前向不再放在 `Dataset.__getitem__()` 里提前计算。
2. 教师特征改为在训练循环中批量前向生成。
3. 学生网络、检测头、冻结教师网络、冻结 response detector 都支持按当前可见 GPU 数自动包装。

这样双卡时，`optical_teacher.py` 不再被数据集阶段的单卡 teacher 前向卡住。

## 十三、相位塌缩修正已并入当前版本

文档前面提到的相位塌缩修正，这次也已经并入当前 `optical_teacher.py`：

1. `NORM_ENABLE_EPOCH` 延后。
2. `LOSS_PHASE_SMOOTH_WEIGHT` 降低。
3. 新增：
   - `LOSS_PHASE_DIVERSITY_WEIGHT`
   - `PHASE_STD_TARGET`
4. 每个 epoch 记录：
   - `PhaseSpan(slm1/slm2)`
   - `PhaseStd(slm1/slm2)`
5. 训练过程中额外保存：
   - `phase_epoch_xxx.png`

因此当前版本不仅支持双卡/单卡自动切换，也比之前更容易直接观察相位是否正在向近常数解塌缩。

## 十二、当前分支已对齐的实现点

当前分支中的 `optical_teacher.py` 已经同步了上面的核心思路，并额外对齐了当前 `optical_teacher_yolo.py` 的 detector 配置：

## 13. 相位塌缩问题记录

在当前 `optical_teacher.py` 的光学学生训练中，通过 `test_Phase_weight.py` 提取到的两层 SLM 相位参数出现了明显的低动态范围现象：

- `slm1.phase_raw` 约为 `[-0.026, 0.035]`
- `slm2.phase_raw` 约为 `[-0.031, 0.032]`
- 总相位跨度仅约 `0.06 rad`

这在当前代码语义下不应视为“正常有效学习结果”。原因是 `SLMLayer.forward()` 中实际参与传播的相位就是 `phase_raw mod 2pi`，因此这里看到的 `phase_raw` 不是无关紧要的小残差，而是实际有效相位的直接来源。若其跨度只有 `0.06 rad`，说明当前学到的是接近常数相位，光学调制能力非常弱。

### 13.1 原因分析

当前分支里相位塌缩更可能来自以下几项共同作用：

1. `LOSS_PHASE_SMOOTH_WEIGHT` 偏大。
   相位平滑项会强烈鼓励相位场变得平坦，早期很容易把相位拉向低起伏解。
2. `student.enable_norm` 开启偏早。
   学生输出做均值归一化后，模型更容易用“弱相位调制 + 归一化补偿”的方式通过损失，而不必真正学出高动态范围波前。
3. 特征拟合与 response map 约束存在局部可用最优。
   只要热区位置和整体响应近似正确，损失就可能下降，但这并不保证相位层学到了足够强的结构化调制。

### 13.2 当前分支已做的修正

为抑制这种“相位越学越平”的趋势，当前分支已在 `optical_teacher.py` 中加入以下修改：

1. 延后归一化开启时机。
   `NORM_ENABLE_EPOCH` 已改到联合阶段后半段，避免学生层在前期尚未学稳时就被归一化压平。
2. 降低相位平滑损失权重。
   `LOSS_PHASE_SMOOTH_WEIGHT` 已从 `0.02` 调整到 `0.005`。
3. 新增相位多样性约束。
   新增：
   - `LOSS_PHASE_DIVERSITY_WEIGHT = 0.05`
   - `PHASE_STD_TARGET = 0.45`

   当两层 SLM 的有效相位标准差过小时，会增加额外惩罚，防止继续向常数相位塌缩。
4. 新增相位统计输出。
   每个 epoch 现在会记录：
   - `PhaseSpan(slm1/slm2)`
   - `PhaseStd(slm1/slm2)`

   这样不需要等训练完成后再手动提取权重，在训练日志里就能直接判断相位是否正在塌缩。
5. 新增相位可视化。
   训练期间会额外生成 `phase_epoch_xxx.png`，同时展示：
   - `slm1/slm2 raw phase`
   - `slm1/slm2 effective phase`

   便于直接观察相位是否仍然接近一张平坦图。

### 13.3 与之前可视化报错的关系

之前另一个分支中出现过：

- `save_feature_comparison()` 里直接对 CUDA tensor 调用 `.numpy()`

导致在可视化阶段报错。当前分支已修正为在函数内部统一先做 `.detach().cpu()`，因此即使外部传入 GPU tensor，也不会再因 numpy 转换触发同类错误。

### 13.4 后续训练时重点观察

下一次在服务器重新训练时，优先看以下信号：

1. 日志中的 `PhaseSpan(slm1/slm2)` 是否仍快速收缩到接近 0。
2. 日志中的 `PhaseStd(slm1/slm2)` 是否长期过低。
3. `phase_epoch_xxx.png` 中两层相位是否仍大面积接近平坦。
4. 学生特征图是否仍主要表现为单一亮斑，而缺少纹理和结构。

若这些问题依然明显，下一步继续优先尝试：

- 进一步降低 `LOSS_PHASE_SMOOTH_WEIGHT`
- 适当提高 `LOSS_PHASE_DIVERSITY_WEIGHT`
- 增加 `PHASE1_STUDENT_EPOCHS`
- 必要时重新设计更强的相位物理先验，而不是只依赖像素级特征拟合

这样后续再用 `Optical_SLM_yolo_model.py` 评估时，不再需要把新 student 特征强行接旧 detector，而是可以直接使用“同一轮训练内共同适配出来”的 student + detector 配对。
