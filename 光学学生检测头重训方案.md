# 光学学生检测头重训方案

## 1. 本次现象

在 `output\OpticalTeacher_enhance_teacher` 的训练结果中，预测几乎不可用。日志显示：

1. `student_only` 阶段后期 `Feature` 基本停在 `0.62` 左右，学生没有继续明显接近教师特征。
2. `student_only` 阶段多次出现 `phase collapse detected`，虽然相位 span 看起来不小，但有效相位 std 长期只有 `0.078` 左右。
3. 进入 `detector_only` 后，检测损失从约 `1.67` 降到约 `0.82`，说明检测头确实能在固定学生输出上继续下降。
4. 进入 `joint_balance` 后，相位 std 从约 `0.49/0.54` 逐步降到约 `0.31/0.34`，说明联合阶段仍在把相位往更平滑、更低变化的方向拉。

这说明问题不只是“相位有没有坍塌”，更关键的是学生输出没有形成一个适合检测头识别的稳定特征空间。

## 2. 主要原因

旧检测头是由 `optical_teacher_yolo.py` 中的教师网络特征训练出来的。它适配的是教师网络输出分布，而不是光学学生相位层输出分布。

即使学生特征在可视化上看起来接近教师特征，它和教师特征仍然可能在以下方面不同：

1. 数值动态范围不同。
2. 前景和背景对比不同。
3. 目标边缘、纹理、局部峰值形态不同。
4. 频域和空间平滑特性不同。

因此旧检测头可以作为参考，但不应该默认作为可训练检测头的初始化目标，更不应该在学生第一阶段里强行用旧检测头 response/proxy 去牵引学生。

## 3. 当前代码调整

本次已将 `optical_teacher.py` 调整为更适合“学生特征 + 新检测头”的路线：

1. 默认输出目录改为 `output\OpticalTeacher_student_detector_relearn`。
2. 新增 `DETECTOR_INIT_MODE = "scratch"`。
3. 可训练检测头默认不加载旧教师检测头权重。
4. 第一阶段 `student_only` 只学习教师特征和相位多样性，不再使用旧检测头 response/proxy。
5. 第二阶段 `detector_only` 冻结学生输出，让检测头从学生特征上重新学习。
6. 第三阶段 `joint_balance` 使用低学生学习率和较高检测头学习率，让学生小幅调整，检测头继续适配。

## 4. 新三阶段建议

第一阶段：`student_only`

目标是让相位层先学出有信息的光学特征，不再被旧检测头牵着走。

第二阶段：`detector_only`

目标是让检测头从学生输出分布上重新学习。这个阶段是当前路线最关键的阶段，不能再假设旧检测头可以直接使用。

第三阶段：`joint_balance`

目标是小幅联合微调。学生学习率应明显低于检测头，避免检测头刚适配好的输入分布再次被学生大幅改掉。

## 5. 后续观察重点

重新训练后重点看：

1. `student_only` 阶段的 feature 图是否出现稳定前景结构。
2. `detector_only` 阶段的检测可视化是否从无框逐渐变成贴近 GT。
3. `joint_balance` 阶段相位 std 是否继续大幅下降。
4. `detector_best.pth` 是否明显优于旧教师检测头直接套学生输出。

如果第二阶段检测损失下降但可视化仍完全无框，下一步应优先检查学生输出的数值分布和检测头输入归一化，而不是继续调旧检测头权重。
